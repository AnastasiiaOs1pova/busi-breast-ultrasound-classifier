# Классификатор УЗИ-снимков молочной железы

Репозиторий содержит учебный проект по классификации УЗИ-снимков молочной железы с использованием transfer learning, интерпретации предсказаний через Grad-CAM и baseline autoencoder для поиска аномалий.

## О проекте

Цель проекта — построить модель для классификации УЗИ-снимков молочной железы на три класса:

- `normal` — норма;
- `benign` — доброкачественное образование;
- `malignant` — злокачественное образование.

Основной акцент сделан не только на обучении модели, но и на полном pipeline работы с медицинскими изображениями:

- подготовка и разбиение датасета;
- аугментация изображений;
- обучение нескольких CNN-моделей с transfer learning;
- оценка качества классификации;
- анализ ошибок модели;
- интерпретация предсказаний с помощью Grad-CAM;
- дополнительный эксперимент с автоэнкодером для поиска аномалий.

## Датасет

В проекте использован датасет BUSI — Breast Ultrasound Images Dataset.

Датасет содержит УЗИ-снимки молочной железы трёх классов:

| Класс | Количество изображений |
|---|---:|
| normal | 133 |
| benign | 437 |
| malignant | 210 |

Всего использовано 780 изображений.

Датасет является несбалансированным: больше всего изображений относится к классу `benign`, меньше всего — к классу `normal`. Чтобы снизить влияние дисбаланса, при обучении использовались веса классов.

## Подготовка данных

На этапе подготовки данных были выполнены следующие шаги:

- приведение изображений к размеру 224×224;
- перевод grayscale-изображений в 3-канальный формат для использования предобученных ImageNet-моделей;
- нормализация изображений;
- разбиение данных на обучающую и тестовую выборки со стратификацией по классам;
- применение аугментаций для обучающей выборки.

Для обучающей выборки использовались следующие аугментации:

- случайное горизонтальное отражение;
- случайный поворот изображения.

## Модели

В проекте были протестированы три предобученные CNN-архитектуры:

- ResNet18;
- MobileNetV3 Small;
- EfficientNet-B0.

Во всех моделях использовался transfer learning: основная сверточная часть модели была взята из предобученной архитектуры, а классификационная голова была заменена под задачу классификации трёх классов BUSI.

## Метрики качества

Для оценки качества моделей использовались следующие метрики:

- accuracy;
- precision;
- recall;
- F1-score;
- macro F1-score;
- weighted F1-score;
- macro ROC-AUC;
- confusion matrix.

Так как датасет несбалансирован, при анализе результатов особенно важны macro F1-score и confusion matrix, а не только accuracy.

## Результаты классификации

Модели показали близкое качество на тестовой выборке. Лучшие результаты по accuracy получили ResNet18 и EfficientNet-B0, а лучший macro ROC-AUC показали MobileNetV3 Small и EfficientNet-B0.

| Модель | Accuracy | Macro F1 | Weighted F1 | Macro ROC-AUC |
|---|---:|---:|---:|---:|
| ResNet18 | 0.769 | 0.741 | 0.769 | 0.883 |
| MobileNetV3 Small | 0.756 | 0.741 | 0.761 | 0.889 |
| EfficientNet-B0 | 0.769 | 0.753 | 0.773 | 0.889 |

По совокупности метрик наиболее стабильный результат показала EfficientNet-B0: при той же accuracy, что и ResNet18, модель получила более высокий macro F1-score и weighted F1-score.

## Интерпретация результатов

Для интерпретации предсказаний использовался Grad-CAM.

Grad-CAM позволяет визуализировать области изображения, которые сильнее всего повлияли на решение модели. 
На отдельных примерах активные зоны Grad-CAM располагались в области образования, что говорит о том, что модель частично ориентировалась на клинически значимые области снимка.

## Автоэнкодер для поиска аномалий

Дополнительно был реализован baseline autoencoder для задачи поиска аномалий.

Автоэнкодер обучался только на изображениях класса `normal`. После обучения модель восстанавливала изображения, а ошибка реконструкции использовалась как anomaly score.

Результаты anomaly detection:

| Метрика | Значение |
|---|---:|
| Precision | 0.932 |
| Recall | 0.318 |
| F1-score | 0.474 |
| ROC-AUC | 0.709 |


В рамках задания автоэнкодер обучался только на изображениях класса `normal`. Порог ошибки реконструкции был выбран по 95-му перцентилю ошибок на нормальных изображениях. Такой подход строит границу нормы: изображения с ошибкой выше порога считаются потенциальными аномалиями.

При выбранном пороге модель показала высокую precision (`0.932`), но низкий recall (`0.318`). Это означает, что автоэнкодер редко ошибочно помечает нормальные изображения как аномальные, но пропускает значительную часть патологических случаев.

Для медицинской задачи такой результат является ограничением: при скрининге важнее находить как можно больше патологий, даже ценой увеличения числа ложноположительных срабатываний. Поэтому автоэнкодер в текущей настройке можно рассматривать только как baseline для anomaly detection, а не как подходящее решение для практического выявления патологий.

## Основные выводы

В проекте был реализован полный pipeline классификации медицинских изображений:

- подготовка и аугментация данных;
- обучение нескольких CNN-моделей с transfer learning;
- сравнение качества моделей;
- анализ метрик на несбалансированном датасете;
- интерпретация предсказаний через Grad-CAM;
- дополнительный baseline autoencoder для поиска аномалий.

Лучшее качество по совокупности метрик показала EfficientNet-B0.

## Ограничения

Основные ограничения:

- небольшой размер датасета;
- дисбаланс классов;
- отсутствие внешней валидации на данных из других клиник и УЗИ-аппаратов;
- возможное влияние артефактов, подписей и различий в качестве изображений;
- модели обучались на отдельных 2D-изображениях без учёта полного клинического контекста пациента;
- Grad-CAM помогает интерпретировать фокус модели, но не доказывает клиническую корректность решения;
- автоэнкодер использовался только как baseline для поиска аномалий.

## Возможные улучшения

В дальнейшем проект можно улучшить следующими способами:

- добавить отдельную валидационную выборку для выбора лучшей модели;
- провести более строгий подбор гиперпараметров;
- использовать более сильные аугментации;
- протестировать дополнительные архитектуры CNN и Vision Transformer;
- провести внешнюю валидацию на другом датасете;
- улучшить обработку дисбаланса классов;
- провести более подробный анализ ошибок по каждому классу;
- сравнить Grad-CAM для правильных и ошибочных предсказаний;
- улучшить anomaly detection с помощью более сложных автоэнкодеров или self-supervised learning.
